# [NewsAPI](https://newsapi.org)

You must sign an account to obtain an API key. There are many subscription plans avaiable. For developing, you can choose "Developer".

> You should only use the Developer plan if your project is in development.
> If your project is being used in production, please upgrade to a paid plan.

## Authentication

There are three ways adding the API key:

1. API with the `apiKey` query string: `GET https://newsapi.org/v2/everything?q=keyword&apiKey={APIKEY}`.
2. Add the [`X-Api-Key`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers) header at the request.
3. Add the [`Authorization`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Authorization) header at the request.

## Parameters

### Common parameters

* ✅: Required
* ✔️: Avaiable
* ❌: Unavaiable

| Endpoints | `apiKey` | `q` | `category` | `language` | `country` | `sources` | `pageSize` | `page` |
| ----------- | ----------- | ----------- | ----------- | ----------- | ----------- | ----------- | ----------- | ----------- |
| `/v2/everything` | ✅ | ✅ (*de facto*) | ❌ | ✔️ | ✔️ | ✔️ | ✔️ | ✔️ |
| `/v2/top-headlines` | ✅ | ✔️ | ✔️ | ❌ | ✔️ | ✔️ | ✔️ | ✔️ |
| `/v2/top-headlines/sources` | ✅ | ❌ | ✔️ | ✔️ | ✔️ | ❌ | ❌ | ❌ |

### `/v2/everything` only parameters

* `domains`: Given domains *only*
* `excludeDomains`: *Not* given domains only
* `from`: ISO 8601 date. Starting date.
* `to`: ISO 8601 date. Ending date.
* `sortBy`: Three values: `relevancy`, `popularity`, `publishedAt`

### Parameter Specs

* `from` and `to`: [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601)
* `language`: [ISO-639-1](https://en.wikipedia.org/wiki/ISO_639-1)
* `country`: [ISO 3166-1](https://en.wikipedia.org/wiki/ISO_3166-1)
